# Investment Document Classifier & Renamer (Python, rule-based)

Automates a manual back-office task from investment operations: taking a pile of
downloaded fund documents, identifying what each one is, finding its date, and
renaming it to a consistent convention before upload.

It replaces the legacy manual process of opening each PDF, reading it, and
hand-renaming it by date, document type, fund, and legal entity.

## How it works
1. You drop the PDFs in a folder.
2. The script converts each PDF to text.
3. It identifies the document type using signature keywords.
4. It detects the document's date.
5. It uses the fund name and legal entity you provide.
6. It proposes a standardized filename and writes a review sheet.
7. You spot-check the review sheet, then re-run with --apply to rename.

Human stays in the loop: nothing is renamed until you pass --apply.

## Document types recognized
- Capital Call Notice
- Capital Account Statement
- Hedge Fund Statement
- Distribution Notice
- K-1 (Schedule K-1 / Form 1065)

Anything it cannot identify is marked "Unknown" and left for manual review.

## Filename convention
`YYYY-MM-DD_DocumentType_FundName_LegalEntity.pdf`

## Why rules, not machine learning
These documents contain reliable signature phrases ("Capital Call Notice",
"Schedule K-1"), so keyword rules are accurate, fully explainable, need no
training data, and a wrong match is easy to trace. For a financial workflow,
explainability matters. (A machine-learning version is provided separately as a
progression, for cases where wording is inconsistent enough that rules struggle.)

## Run it
```bash
# preview only (writes a review sheet, renames nothing)
python classify_documents.py --folder ./sample_pdfs --fund "Evergreen Capital Partners III" --entity "Evergreen GP LLC"

# after checking the review sheet, actually rename
python classify_documents.py --folder ./sample_pdfs --fund "Evergreen Capital Partners III" --entity "Evergreen GP LLC" --apply
```
If you omit --fund or --entity, the script prompts for them.

## Output
- classification_review.csv  in the folder, listing each file, its detected
  type, confidence, date, proposed name, and any review notes.
- Renamed files, only when --apply is used.

## Notes on accuracy
- Confidence is the number of matched keywords. A score of 1 is flagged as low
  confidence for you to check.
- K-1s often carry only a tax year, not a date. In that case the date is set to
  that year's December 31 and flagged as "inferred" so you know it was derived.
- The review-then-apply split means a misread never silently renames a file.

## Dependencies
- pdfplumber  (PDF text extraction):  pip install pdfplumber
Scanned/image-only PDFs would additionally need OCR (pytesseract); the included
samples are text-based and do not require it.

## Test data
The sample_pdfs folder contains seven fictional investment documents covering
all five types, including edge cases, generated by make_samples.py. All names
and figures are invented.
